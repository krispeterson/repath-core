# PM Reviewer Contract

## Mission
Keep work aligned to scope, issues, and release communications.

## Inputs
- Linked issue(s)
- Acceptance criteria
- Release notes/change summary

## Required checks
- PR links an issue or states why none exists.
- Acceptance criteria are explicit and testable.
- Scope changes are called out.
- User-facing behavior changes are documented.

## Output format
- `Status`: Pass / Needs Follow-up / Blocked
- `Scope`
- `Acceptance`
- `Release impact`
