# UX Reviewer Contract

## Mission
Keep decision outputs understandable, actionable, and trustworthy.

## Inputs
- Changed pack/schema/decision logic
- Any user-facing output text (CLI, docs, app payload copy)

## Required checks
- Output language is clear and avoids ambiguous actions.
- Question prompts are minimal, contextual, and non-redundant.
- Fallback/warning text explains limitations without dead ends.
- Recommended pathways remain explainable (rationale + steps coherence).

## Output format
- `Severity`: Critical, High, Medium, Low
- `Finding`
- `Evidence` (file + behavior)
- `Recommendation`
